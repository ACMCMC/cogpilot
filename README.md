# CogPilot: Reinventing the Wheel with Preventative Drowsiness Detection

**Hackathon Project:** Android + Snowflake Native AI for Active Driver Safety

> **The "Reinvention":** Traditional drowsiness detection is **reactive** (beeps when eyes close) and **passive**. CogPilot is **preventative** and **active**вҖ”it predicts monotony fatigue using a hybrid risk model and triggers cognitive stimuli (debates, puzzles, trivia) generated by Snowflake Arctic before the driver falls asleep.

---

## рҹҺҜ Project Overview

CogPilot is a full-stack mobile system that prevents driver drowsiness by:

1. **Monitoring** real-time GPS telemetry (speed, heading, road monotony)
2. **Predicting** drowsiness risk using a hybrid model (Monotony + Time-on-Task - Road Complexity)
3. **Intervening** with AI-generated cognitive stimuli **before** fatigue occurs
4. **Engaging** the driver directly with Snowflake Arctic + Cortex

### Key Features
- вң… **Material Design 3 UI** - Modern Android interface optimized for driving
- вң… **Direct Snowflake Integration** - JDBC connection, no backend needed
- вң… **Snowflake Cortex** - `CORTEX.COMPLETE('snowflake-arctic')` for AI stimulus generation
- вң… **Hybrid Risk Scoring** - Preventative model (monotony + time-on-task - complexity)
- вң… **Real-time Telemetry** - 5-second GPS polling, speed/heading variance tracking
- вң… **Driving Mode Support** - Works while vehicle is in Drive gear

## рҹ“Ҡ Risk Scoring Algorithm

**Hybrid Risk Model:**

```
Risk Score = (0.4 Г— Monotony) + (0.3 Г— TimeOnTask) - (0.3 Г— RoadComplexity)

Where:
  вҖў Monotony = 1 - (stddev(speed) / 2.0)           [0-1]
  вҖў TimeOnTask = time_since_intervention / 1800000 [0-1]
  вҖў RoadComplexity = heading_variance / 45В°        [0-1]

Intervention Threshold: Risk > 0.6
```

**Examples:**

| Scenario | Speed Variance | Heading Variance | Time | Risk | Status |
|----------|---|---|---|---|---|
| Highway monotony | Low (60 mph) | Low (90В°) | 25min | **0.71** | рҹҡЁ INTERVENE |
| City driving | High (varies) | High (turns) | 10min | 0.18 | вң… SAFE |
| Light traffic | Medium | Medium | 15min | 0.42 | вҡ пёҸ CAUTION |

## рҹ§  Risk Scoring Algorithm (The Core "Reinvention")

### Hybrid Risk Window Model

CogPilot predicts drowsiness **before** it happens by combining three signals:

#### 1. **Monotony Factor** (40% weight)
- **What:** Standard deviation of speed over last 2 minutes
- **Why:** Highway driving = low variance = drowsy conditions
- **Calculation:**
  ```
  std_dev = stdev(speeds_last_2_min)
  if std_dev < 2.0 mph:
    monotony = 1.0 - (std_dev / 2.0)  # HIGH
  else:
    monotony = 0.0                    # LOW
  ```

#### 2. **Time-on-Task Factor** (30% weight)
- **What:** Minutes since last intervention
- **Why:** Longer drive without engagement = more fatigue accumulation
- **Calculation:**
  ```
  time_factor = min(time_since_intervention / 1800, 1.0)
  # Max accumulation at 30 minutes without break
  ```

#### 3. **Road Complexity Factor** (30% weight, SUBTRACTED)
- **What:** Standard deviation of heading (degrees)
- **Why:** Turns = active driving = suppress intervention (driver is alert)
- **Calculation:**
  ```
  heading_variance = max(headings) - min(headings)
  complexity = min(heading_variance / 45.0, 1.0)
  # High variance (turns) = high complexity = suppress intervention
  ```

#### Final Formula
```
Risk_Score = (0.4 Г— monotony) + (0.3 Г— time_factor) - (0.3 Г— complexity)
Range: 0.0 (safe) to 1.0 (critical)
Threshold: Risk_Score > 0.6 вҶ’ INTERVENE
```

#### Example Scenarios

**Scenario A: Highway monotony вҶ’ HIGH RISK**
- Speed: 55 mph Вұ1 mph (std dev = 0.8)
- Heading: 90В° Вұ2В° (variance = 4В°, low turns)
- Time since intervention: 25 min

```
Monotony = 1.0 - (0.8/2.0) = 0.6
Time = 25/30 = 0.83
Complexity = 4/45 = 0.09

Risk = (0.4 Г— 0.6) + (0.3 Г— 0.83) - (0.3 Г— 0.09)
     = 0.24 + 0.25 - 0.03
     = 0.46 (MONITOR)
```

**Scenario B: Extended highway + time accumulation вҶ’ INTERVENE**
- Speed: 55 mph Вұ0.5 mph (std dev = 0.3)
- Heading: 90В° Вұ1В° (variance = 2В°, straight road)
- Time since intervention: 35 min

```
Monotony = 1.0 - (0.3/2.0) = 0.85
Time = min(35/30, 1.0) = 1.0
Complexity = 2/45 = 0.04

Risk = (0.4 Г— 0.85) + (0.3 Г— 1.0) - (0.3 Г— 0.04)
     = 0.34 + 0.30 - 0.01
     = 0.63 вҶ’ TRIGGER INTERVENTION! рҹҡЁ
```

## рҹ“Ҡ Database Schema

### DRIVER_LOGS
```sql
CREATE TABLE DRIVER_LOGS (
    id INTEGER AUTOINCREMENT,
    timestamp BIGINT NOT NULL,           -- Epoch milliseconds
    speed FLOAT NOT NULL,                 -- mph
    heading FLOAT NOT NULL,               -- degrees (0-360)
    latitude FLOAT NOT NULL,
    longitude FLOAT NOT NULL,
    user_text VARCHAR,                    -- Transcribed driver speech
    sentiment_score FLOAT,                -- Result of CORTEX.SENTIMENT
    risk_score FLOAT,                     -- Calculated hybrid risk score
    created_at TIMESTAMP_NTZ DEFAULT CURRENT_TIMESTAMP(),
    PRIMARY KEY (id)
);
```

### USER_PROFILE
```sql
CREATE TABLE USER_PROFILE (
    driver_id INTEGER PRIMARY KEY,
    interest_topics VARCHAR,              -- CSV: "Physics,90s Rock,Philosophy"
    complexity_level VARCHAR,             -- basic, intermediate, advanced
    created_at TIMESTAMP_NTZ DEFAULT CURRENT_TIMESTAMP()
);
-- Default: driver_id=1, interests=[Quantum Physics, 90s Rock, Philosophy]
```

### COGNITIVE_STIMULI
```sql
CREATE TABLE COGNITIVE_STIMULI (
    id INTEGER AUTOINCREMENT,
    stimulus_type VARCHAR NOT NULL,       -- debate, trivia, logic_puzzle
    prompt VARCHAR NOT NULL,              -- The prompt sent to Arctic
    response VARCHAR,                     -- Arctic's generated response
    audio_url VARCHAR,                    -- URL to TTS MP3 (optional)
    generated_at TIMESTAMP_NTZ DEFAULT CURRENT_TIMESTAMP(),
    PRIMARY KEY (id)
);
```
